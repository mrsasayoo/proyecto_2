### Task A2: Refactorizar `download_pancreas()` para multi-batch

**Files:**
- Modify: `src/pipeline/fase0/descargar.py:370-384`

**Cambio exacto** — reemplazar la función entera:

```python
def download_pancreas(datasets_dir, batches=None):
    # type: (Path, list|None) -> bool
    """Descarga uno o más batches de PANORAMA CT Pancreas desde Zenodo.

    Args:
        datasets_dir: directorio base de datasets
        batches: lista de enteros [1..4]. Por defecto [1] (compatibilidad).
    """
    if batches is None:
        batches = [1]
    all_ok = True
    for batch_num in batches:
        info = ZENODO_PANCREAS_BATCHES.get(batch_num)
        if info is None:
            log.error("[PANCREAS] Batch %d no reconocido (válidos: 1-4).", batch_num)
            all_ok = False
            continue
        dest = datasets_dir / "zenodo_13715870" / "batch_{}.zip".format(batch_num)
        ok = download_wget(
            info["url"],
            dest,
            "PANCREAS-B{}".format(batch_num),
            extra_flags=[
                "--tries=5",
                "--retry-connrefused",
                "--waitretry=30",
                "--timeout=120",
            ],
            min_size_bytes=MIN_ZIP_SIZES["pancreas"],
        )
        if not ok:
            all_ok = False
    return all_ok
```

```markdown
#### LUNA16 — Normalización HU + ImageNet: no es doble normalización

La cadena `normalize_hu()` → `volume_to_vit_input()` es **intencional y correcta**:
1. `normalize_hu(-1000, 400)` → lineal [0, 1] **offline** (almacenado en `.npy`)
2. `(x - μ_ImageNet) / σ_ImageNet` → z-score **online** en `__getitem__`

El paso 2 es necesario porque el backbone ViT fue pre-entrenado en ImageNet y espera
esa distribución de entrada. Aplicar z-score sobre datos ya en [0, 1] es el protocolo
estándar para adaptar imágenes médicas a ViTs pre-entrenados. No es una redundancia.
**Estado: NO-ISSUE — cerrado.**
```
